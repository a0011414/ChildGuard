# Xcode から実機で Run するとき「開発者を信頼する」手順

「アプリを検証できません」「The Developer App Certificate is not trusted」と出る場合、**iPhone 側で「この開発者を信頼する」操作が必要**です。

**重要（iOS 16 以降）**: 「VPNとデバイス管理」に開発者用の行が**まったく出ない**ときは、**Developer Mode（デベロッパモード）を先にオン**にしてください。オフのままだと信頼する項目が表示されないことがあります。

---

## 最新の仕様（Xcode 26 / iOS 26 時点）

- **Xcode 26**（公式ドキュメント）では、実機・シミュレータの管理は **Window > Devices and Simulators** で開く「デバイスとシミュレータ」ウィンドウで行います。日本語 Xcode では **ウィンドウ** メニュー → **デバイスとシミュレータ**（または類似の名前）です。
- **「ウィンドウ」メニューが見つからない場合**: メニューバーは**いま前面にあるアプリ**のものです。**Cursor やブラウザを開いたまま**だと、メニューは Cursor の「ファイル」「編集」…になります。**Xcode をクリックして前面にし**、メニューバーが Xcode の表示（Xcode, ファイル, 編集, 表示, … **ウィンドウ**, ヘルプ）になってから、**ウィンドウ** を開いてください。ショートカット **Shift + Command + 2** で Organizer が開き、その中の **Devices** タブで実機を選べる場合もあります。
- **iOS 26**: 「開発者を信頼」の項目が **VPNとデバイス管理 に表示されない**事例が複数報告されています。Developer Mode をオンにしていても、証明書の行が出ないことがあります。Apple の公式手順では「Settings > General > VPN & Device Management で開発者を選択して Trust」とされていますが、環境によっては表示されず、現時点で確実な対処は TestFlight 経由のインストールです（下記 F 参照）。

---

## 「デバイスとシミュレータ」で何を見るか（iPhone を左で選んだあと）

左で **自分の iPhone** を選ぶと、**右側のエリア**にその端末の情報が出ます。次を確認してください。

| 見る場所 | 何が書いてあるか |
|----------|------------------|
| **右側のいちばん上** | **黄色やオレンジの枠**で警告が出ていることがあります。「Trust this computer」や「開発者を信頼してください」など、**やるべきことが短い文で書いてある**ので、その通りにするとよいです。 |
| **右側のボタン** | **「Prepare for Development」**（開発の準備）や **「Trust」** などのボタンがある場合、それを押すと Xcode が端末の準備を進めます。押したあと、iPhone に「このコンピュータを信頼しますか？」と出たら **信頼** をタップしてください。 |
| **右側の「Status」や状態** | **「Ready」** なら準備完了。**「Preparing」** ならしばらく待つ。**「Unavailable」** や **「Unsupported」** なら、OS のバージョンや接続に問題がある可能性があります。 |
| **右側の「Signed in as」** | この端末で開発に使っている **Apple ID** が表示されます。ここに自分のアカウントが出ていれば、その端末向けのプロファイルが Xcode で使われます。 |

**右側の上に黄色い警告やボタンが無く、Status が Ready のまま**なのに、実機で「アプリを検証できません」と出る場合は、iPhone 側の「VPNとデバイス管理」に開発者用の行が出ていない可能性が高く、その場合は **TestFlight でインストール**（下記 F）が確実です。

---

## 手順 0：Developer Mode をオンにする（行が 1 つも出ない人向け）

1. **設定** → **プライバシーとセキュリティ**（英語なら Privacy & Security）を開く。
2. 下の方の **「デベロッパモード」**（Developer Mode）を探す。
3. **オン**にする。再起動やパスコード入力を求められたら従う。
4. 再起動後、次の「手順 1」から進む。

※ Developer Mode が一覧に無い場合は、iPhone を Mac に USB でつないだ状態で Xcode を開き、1 回 Run を試してから設定を見直すと出ることがあります。

---

## 手順（iOS 26 や新しい OS でも使えるように）

### 1. まず Xcode で 1 回 Run する

- iPhone を USB で接続し、Xcode で **Run（⌘R）** を実行する。
- アプリはインストールされるが、**起動に失敗**して「検証できません」などと出る。**そのままでよい。**
- このタイミングで、iPhone に「開発者用のプロファイル」が登録される。

### 2. iPhone で「設定」を開く

- **設定** → **一般** → **VPNとデバイス管理**（または **デバイス管理**）を開く。

### 3. 開発者を探す（表示は機種・OS で違います）

**「VPNとデバイス管理」の画面で、次のどれかを探してください。**

| 探すもの | 説明 |
|----------|------|
| **デベロッパ App** | 見出しの下に **1 行**（メールアドレスやチーム名）がある。それをタップする。 |
| **デバイス管理** | 見出しをタップすると、中に **開発者やメールアドレス** が 1 つ出る。それをタップする。 |
| **プロファイル** | 「VPNとデバイス管理」の**上**（一般の一覧）に **プロファイル** がある場合、そこを開き、開発者用のプロファイルをタップする。 |
| **何か 1 行** | 「Apple Development: xxx@mail.com」やチーム名だけの行が 1 つある場合、それをタップする。 |

- **下までスクロール**してみる。項目が下の方にあることがあります。
- **英語表示なら**: "Developer App" や "Device Management" の下に、あなたの **メールアドレス（Apple ID）** が 1 行で出ていることが多いです。

### 4. 「信頼」をタップする

- 開発者（メールアドレスやチーム名）をタップすると、画面が開く。
- **「○○を信頼」**（または **「信頼」**）のボタンをタップする。
- 確認で **「信頼」** をもう一度タップする。

### 5. アプリを起動する

- **ホーム画面** に戻り、**ChildGuard** のアイコンをタップして起動する。  
  または **Xcode で再度 Run（⌘R）** してもよい。

## それでも「信頼する項目」が 1 つもない場合

### C. ペアリングをリセットしてからやり直す

開発用のペアリングが壊れていると、証明書が端末に登録されないことがあります。

1. **設定** → **一般** → **iPhoneを転送またはリセット** → **リセット** を開く。
2. **「位置情報とプライバシーをリセット」** をタップする（iPhone のデータは消えません。位置情報・プライバシー設定が初期状態に戻るだけです）。
3. リセット後、**iPhone を USB で Mac に再接続**する。
4. Xcode で **Run（⌘R）** を実行する。
5. インストールが終わったら、**設定** → **一般** → **VPNとデバイス管理** を開き、開発者用の行が出ていないか確認する。

### D. iPhone の言語を一時的に英語にする  
  **設定** → **一般** → **言語と地域** → **iPhone の使用言語** を **English** に変更する。  
  すると「VPN & Device Management」の下に **"Developer App"** と **あなたのメールアドレス** が表示されやすくなります。信頼したあと、必要なら言語を日本語に戻してよいです。

### E. アプリを削除してからやり直す

1. iPhone で ChildGuard を削除する。  
2. Xcode で再度 **Run（⌘R）** してインストールする。  
3. **インストールが終わった直後**に、**まだアプリを開かず**、**設定 → 一般 → VPNとデバイス管理** を開く。  
4. 新しく出た 1 行（開発者）をタップ → **信頼**。

### F. TestFlight でインストールする（開発者を信頼する必要なし）

**Xcode から直接 Run できなくても、実機でアプリを試す方法**です。TestFlight 経由で入れたアプリは「開発者を信頼」しなくても起動できます。

1. Mac で Xcode から **Product → Archive** でアーカイブを作る。
2. **Distribute App** で **App Store Connect** にアップロードする。
3. App Store Connect の **TestFlight** タブで、自分（またはテスト用のメール）を **内部テスト** に追加する。
4. iPhone に **TestFlight** アプリを入れる（まだなら App Store からインストール）。
5. 招待メールまたは TestFlight アプリ内で **ChildGuard** を **インストール**する。

くわしくは **TestFlight手順.md** を参照してください。

---

## まとめ

1. Xcode で 1 回 Run → インストールされる（起動失敗でよい）。
2. **設定 → 一般 → VPNとデバイス管理** を開く。
3. 表示された **開発者（メールアドレスやチーム名）** をタップ → **「信頼」**。
4. ChildGuard を起動する。

これで「アプリを検証できません」は解消されます。
